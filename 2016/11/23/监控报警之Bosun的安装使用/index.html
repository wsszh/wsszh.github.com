监控报警之Bosun的安装使用

发表于 2016-11-23   |   分类于 监控管理   |     |   阅读次数

Bosun是由Stack Exchange团队开发的一套基于时间序列的开源的监控报警系统，可以与OpenTSDB结合在一起，在OpenTSDB的基础上增加了报警的功能
官方文档的QuickStart中是以docker安装方式进行说明的，官方提供的docker镜像包含了OpenTSDB和HBase
我已经事先装好了OpenTSDB和HBase，暂时就先不用docker安装，又因为Bosun是用go开发的，而系统还没有安装go，所以我这里选择用官方提供的二进制版本进行安装

添加自定义监控脚本

scollector和tcollector一样，也支持自定义的监控脚本（语言不限，只要是有可执行权限的程序），只需要在stdout上按以下格式输出：

Metrics Timestamp Value [Tags]

  • Metric 是对应metric的名称
  • Timestamp 是Unix时间戳，以秒为单位
  • Value 是数据点的值，浮点型
  • Tags 是可选的，名为host的tag会自动添加，但若输出中有host这个tag，就会覆盖自动添加的值

选择一个metric

我们点击Bosun页面上的Items标签，可以通过输入框搜索快速找到 linux.cpu 这个metric

然后页面会跳转至 linux.cpu 对应的Graph页面，可以输入主机名，获得某个特定主机的数据，系统默认开始时间为 1h-ago，所以点击 Query按钮后，我们可以看到由过去一小时内的数据绘制而成的曲线图：

在这个页面的底部右下角，分别有Expression和Rule两个标签

  • Expression可以跳转至Expression页面，我们可以在里面修改并测试查询表达式
  • Rule可以跳转至Rule Editor页面，我们可以在里面设置并测试具体的报警规则
