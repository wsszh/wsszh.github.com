本地Ubuntu 16.04虚拟机中利用kubeadm安装kubernetes v1.10.0集群

发表于 2018-08-08   |   分类于 云计算   |     |   阅读次数

第一次配置Kubernetes环境，坑还是挺多的，做个记录.

准备工作

  • Ubuntu 16.04 64位系统的虚拟机至少2台.

  • 有科学上网的能力.

  • 下文中所有的安装包以及镜像都从这里下载，密码: aesw.

系统环境

简便起见，暂时先只配置两个节点.

hostname IP
master 192.168.190.128
slave1 192.168.190.129

安装软件

以下操作需要在所有的master和slave1节点上执行.

安装依赖

安装 docker，kubeadm，kubectl，kubelet

如果有缺少依赖的情况，可以使用 apt-get -f install 命令修复依赖.

系统设置

以下操作需要在所有的master和slave1节点上执行.

关闭swap

不过这个命令重启后就会失效，需注意.

关闭防火墙

初始化k8s集群

坑1：初始化前，必须确保终端是可以科学上网的，可以用 curl www.google.com 验证，具体实现参考Linux设置命令行代理，利用 polipo 转 socks 为 http. 否则执行下面的初始化命令后，会一直卡在 [init] This might take a minute or longer if the control plane images have to be pulled.

利用 kubeadm 初始化集群:

坑2：必须加 --kubernetes-version=v1.10.0 这个参数，不然 kubeadm 会默认先去联网确认 Kubernetes 的版本(这一操作要科学上网，就算能科学上网，网速慢那也是浪费时间).

成功执行后，需要记录最后一行形如 kubeadm join ... 的输出信息，后面 slave1 节点加入集群时会用到.

然后根据输出信息中的提示，分别执行以下三句命令：

然后可以执行 sudo kubectl get pods --all-namespaces 命令查看 pod 信息:

可以看到 kube-dns 的 READY 状态为 0/3，这是因为 kube-dns 默认跑在 slave1 节点的. 用 kubectl describe 命令查看详细描述找到原因.

slave1节点加入集群

在 slave1 节点执行 sudo kubeadm join ... 命令，即上面记录的在成功初始化 master 节点后输出的最后一行.

然后在 master 节点用 sudo kubectl get nodes 命令查看 node 信息:

可以看到STATUS都是NotReady，这是因为还未安装pod network，官方教程中说明了这一点:

You MUST install a pod network add-on so that your pods can communicate with each other.

The network must be deployed before any applications. Also, kube-dns, an internal helper service, will not start up before a network is installed. kubeadm only supports Container Network Interface (CNI) based networks (and does not support kubenet).

安装 pod network

在 master 节点执行命令

修改 calico.yaml 文件，将 CALICO_IPV4POOL_CIDR 的 value 修改为 10.244.0.0/16(CALICO_IPV4POOL_CIDR 的值必须与初始化 master 节点时的 --pod-network-cidr 值保持一致).

坑3：这里还需要查看 calico.yaml 文件中 quay.io/coreos/etcd:v3.1.10, quay.io/calico/node:v3.0.4, quay.io/calico/cni:v2.0.3, quay.io/calico/kube-controllers:v2.0.2 等对应的版本是否与之前加载的镜像的版本一致，不一致则修改.

然后执行 sudo kubectl apply -f calico.yaml 安装 pod network.

执行 ifconfig 命令后如果看到多了一个 tunl0 网卡，且IP符合 10.244.0.0/16 这个网段，则安装成功，slave1 节点同理.

此时再执行 kubectl get pods --all-namespaces 可以看到 kube-dns 变成了 READY:3/3，正常启动了:

执行 sudo kubectl get nodes 可以看到 master 和 slave1 节点的 STATUS 都变成了 Ready.
